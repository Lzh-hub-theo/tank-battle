# 一、问题建模（核心第一步）

## 1. 场景设定

* 环境：二维栅格地图（Grid World）
* Agent：从起点移动到终点
* 约束：

  * 障碍物不可穿越
  * 每次移动一步

## 2. MDP建模

定义：

* 状态 (S)：当前位置 ((x, y))
* 动作 (A)：{上、下、左、右}
* 奖励 (R)：

  * 到达终点：+100
  * 撞墙/障碍：-100
  * 普通移动：-1
* 折扣因子 (\gamma)：0.9～0.99

---

# 二、环境设计（Environment）

## 1. 地图表示

* 二维数组：

  ```
  0 = 可通行
  1 = 障碍
  G = 目标
  S = 起点
  ```

## 2. 状态编码

* 方式1：直接用 (x, y)
* 方式2：映射为整数 state_id（推荐）

## 3. 状态转移逻辑

* 执行动作 → 更新位置
* 判断：

  * 是否越界
  * 是否撞障碍
  * 是否到达终点

---

# 三、Q-learning算法设计（核心）

Q值更新公式（关键）👇

Q(s,a) \leftarrow Q(s,a) + \alpha \big[r + \gamma \max_{a'} Q(s',a') - Q(s,a)\big]

参数说明：

* 学习率 (\alpha)：0.1～0.5
* 折扣因子 (\gamma)：未来奖励权重
* (r)：即时奖励

---

# 四、Q表初始化

## 1. Q-table结构

* 维度：

  ```
  Q[state][action]
  ```
* 初始化为：

  * 全0 或 小随机值

## 2. 状态空间大小

* 若地图为 N×M：

  * 状态数 = N × M
  * 动作数 = 4

---

# 五、训练流程（核心流程）

## 1. 主循环（Episode）

重复多轮训练：

```
for episode in range(N):
    初始化起点
    while 未到终点:
        选择动作
        执行动作
        获取奖励
        更新Q值
```

## 2. 动作选择策略（探索 vs 利用）

### ε-greedy策略：

* 以 ε 概率随机探索
* 以 1-ε 选择最优动作

建议：

* ε 从 1 → 0.1 逐渐衰减

---

# 六、奖励函数设计（成败关键）

推荐设计：

| 行为    | 奖励       |
| ----- | -------- |
| 到达目标  | +100     |
| 撞障碍   | -100     |
| 每移动一步 | -1       |
| 靠近目标  | +1（可选优化） |

👉 技巧：

* 避免“绕圈”：增加步数惩罚
* 加速收敛：加入“距离变化奖励”

---

# 七、终止条件

每个 episode 结束条件：

* 到达终点 ✅
* 步数超过上限（防止死循环）

---

# 八、路径生成（推理阶段）

训练完成后：

## 1. 固定策略

* 从起点出发
* 每步选择：

  ```
  argmax(Q[state])
  ```

## 2. 输出路径

* 记录路径序列：

  ```
  [(x1,y1), (x2,y2), ...]
  ```

# 十一、常见问题与优化

## 1. 不收敛

* 调整：

  * 学习率 α
  * 折扣因子 γ
  * ε 衰减策略

## 2. 走弯路

* 增加距离奖励
* 加强步数惩罚

## 3. 状态空间过大

👉 Q-learning缺点（重点！）

* 表格爆炸（curse of dimensionality）

解决：

* 状态离散化
* 或升级为 DQN（深度强化学习）
